# Gerenciador de Tarefas Ágil

## Descrição do Projeto
Este projeto consiste no desenvolvimento de um sistema simples de gerenciamento de tarefas, utilizando conceitos da Engenharia de Software e metodologias ágeis. O sistema foi desenvolvido pela empresa fictícia TechFlow Solutions para atender uma startup do setor de logística.

O objetivo do sistema é permitir o acompanhamento do fluxo de trabalho em tempo real, a priorização de tarefas críticas e o monitoramento das atividades da equipe.

---

## Objetivo
Desenvolver um sistema funcional que simule um ambiente real de desenvolvimento ágil, utilizando o GitHub como ferramenta central para versionamento, organização de tarefas, controle de qualidade e gestão de mudanças.

---

## Escopo do Projeto

### Escopo Inicial
- Criar tarefas
- Listar tarefas
- Atualizar tarefas
- Excluir tarefas
- Organizar o fluxo de trabalho

### Mudança de Escopo
Durante o desenvolvimento, foi identificada a necessidade de adicionar o controle de prioridade das tarefas (baixa, média e alta), permitindo melhor organização das atividades críticas da equipe. Essa mudança foi registrada neste documento e no quadro Kanban do projeto.

---

## Metodologia Utilizada
Foi adotada a metodologia ágil Kanban, utilizando o GitHub Projects para o planejamento e acompanhamento das tarefas. O quadro Kanban foi estruturado com as seguintes colunas:
- A Fazer
- Em Progresso
- Concluído
